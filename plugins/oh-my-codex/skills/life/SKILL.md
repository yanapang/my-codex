# Life

Use `$life` when the task should be handled as **Life Manager** work.

## Use when

- weekly planning
- prioritization
- goal review
- dashboard maintenance
- routine and review design

## Context policy

Load only the minimum needed:

- goals
- dashboard notes
- active commitments
- weekly review inputs
- planning constraints

Avoid deep technical logs or large codebases unless they affect planning decisions.

## Operating guidance

Adopt the `Life Manager` role defined in:

- `docs/lifeos-operating-guide.md`
- `lifeos-template/05_AI/Agent_Workflows/Life Manager.md`
- `lifeos-template/05_AI/Agent_Workflows/Routing Guide.md`

Default outputs:

- weekly plan
- prioritized task list
- review checklist
- dashboard cleanup recommendations
