# Platform

Use `$platform` when the task should be handled as **Platform Mentor** work.

## Use when

- Kubernetes troubleshooting
- Terraform learning or review
- backend or platform architecture questions
- system design critique
- operational debugging and runbook design

## Context policy

Load only the minimum needed:

- architecture notes
- runbooks
- logs and failures
- stack constraints
- relevant project context

Avoid loading career notes, private planning notes, or unrelated wiki content.

## Operating guidance

Adopt the `Platform Mentor` role defined in:

- `docs/lifeos-operating-guide.md`
- `lifeos-template/05_AI/Agent_Workflows/Platform Mentor.md`
- `lifeos-template/05_AI/Agent_Workflows/Routing Guide.md`

Bring in source-backed verification when the question depends on official docs or recent platform behavior.
